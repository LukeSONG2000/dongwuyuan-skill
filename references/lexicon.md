## 核心人物原话

### 电棍 / 侯国玉 / otto

| 原话或定型 | 含义与用法 | 可用变体 | 分寸 |
|---|---|---|---|
| 无所谓的，寄了 | 操作失败或局势无望后的摆烂收尾 | `无所谓的，崩了`、`寄了寄了` | 很泛用 |
| 一二级没处理好 | 给失败找一个看似专业但局部的理由；常与“寄了”连用 | `第一步没处理好` | 自嘲比甩锅自然 |
| 怎么到你这儿就拉了胯了呀 | 完整的拉胯责问节奏 | 替换“你这儿”和问题项 | 不针对弱势对象 |
| 你老想着刷你那B三狼干什么玩意啊 | 吐槽执着于次要目标、错失关键时机 | `你老惦记着你那X干什么` | `B`可删以适应正式环境 |
| 打野的走位我就觉得离谱 | 复盘时把问题归给关键角色的位置或决策 | `这个接口的走位离谱` | 适合戏仿甩锅 |
| 急一波不行啊 | 为上头或冒险操作辩解 | `改一版不行啊` | 多为反讽 |
| 我觉得我是 | 对身份/资格给出犹豫又自信的回答 | `你是专家吗？我觉得我是` | 保留停顿感更像 |
| 我喜欢你，你喜欢我 | 把复杂表达压缩成朴素结论，也可无意义直球回复 | 替换两端人物/事物 | 可真诚也可抽象 |
| 那我缺的营养这一块子谁给我补啊 | 对遗漏、缺货、损失进行夸张追责 | `我缺的睡眠谁给我补` | 适合生活吐槽 |
| 吹唢呐 / 嘟嘟嘟 | 用拟声制造送走、结束、鬼畜感 | 只保留`嘟嘟嘟` | 原始辱骂家属内容不要复现 |
| S6第一个王者 | 反复强调旧日成就或资历 | `项目第一个绿灯的人` | 可配“含金量” |
| 含金量 | 讨论成就、版本或环境是否值钱，也可反串 | `这波含金量还在上升` | 很泛用 |
| 203954 | S6登顶截图中的倒计时密码式符号 | 通常原样作彩蛋 | 只给熟悉受众 |
| 第N把好手 | 夸某人对第N擅长项拿手 | `表格第三把好手` | 轻梗 |
| 血魔流 / 爆杀流 | 夸张命名回血续航或瞬间高输出方案 | `血魔流加班`、`爆杀流重构` | 游戏/技术语境好用 |
| 下播之刃 | 某选项一用就导致结束、破防或退出 | `下班之刃` | 套用“X之刃” |
| 韩式/荷式运营 | 在劣势下靠单带、偷发育扳回 | `工位运营` | “荷”相关旧地域影射不要展开 |
| 开庭 | 输后逐帧复盘、分锅、举证 | `日志开庭`、`先开庭再判锅` | 很适合技术复盘 |
| 大X特X | 把某动作做到极致 | `大摆特摆`、`大创特创`、`大调特调` | 高泛用模板 |
| X放不下Y | 形容难割舍 | `他都放下职业生涯了，你还放不下旧接口` | 避免过长解释 |

## 主页面相关梗

### 吉吉国与二创

- **吉吉国王**：电棍因猴相、金发及姓名联想获得的称号；“猴/国王/国民/将军”构成二创世界观。
- **海绵宝宝X儿**：仿电棍旧微博名的昵称格式。
- **稳健棍/移动防御塔**：形容比赛中极稳、难抓；可变为“移动防火墙”。
- **轮椅/♿**：历史上是电棍二创视觉符号。涉及真实残障时不要使用。
- **虚空卡比兽说**：给传话或挑衅套一个转述者；不要借此传播真实谣言。
- **小炮跳脸**：主动跳进危险位置并瞬间暴毙；可泛指冒进部署。
- **猪B吧/你要这么蠢就别玩X**：攻击性原话。仅理解，不对真实用户复现；可降级为“这操作多少有点着急”。
- **V5 G8**：问“缺X吗”后的粗口谐答；默认不用在公开输出。
- **什么之魂**：对关键词咬钩、反复追问；可写`什么之魂呀主播？`。
- **相遇天使**：二次元点歌兼用句；仅相关语境。
- **活字印刷**：把原话拆字重组为新台词；生成文字时可模拟断句，不要声称是真实原话。
- **鬼叫/豪放派/婉约派**：高密度突脸叫声与温和采样的二创分类；可用于描述作品风格。
- **栗子头/说的道理/说的道莉/说的老理**：电棍脸变形衍生形象族；`栗理同源`表示不同形象源自同一原型。
- **棍母不存在**：二创中以“空白/不存在”作为设定。涉及现实家属时不要延伸。

### 哈姆

- 阿拉伯语幼教动画小人，因低清、拍手跳舞、重复呼喊而显得诡异又洗脑。
- 常作无意义的古神/外神、突脸或结尾 jumpscare；“没什么意义就是意义”是理解关键。
- 可用`哈姆哈姆`制造荒诞结尾，或把意义不明的重复话称为“哈姆语”。不要凭空伪造阿拉伯语翻译。

### 别急熊

- 来自 Walking Polar Bear，`北极熊`谐成`别急熊`。
- 用于劝人别急、给焦虑场面配慢悠悠行走的反差。
- `秦始皇骑北极熊`是后期跨圈变体，强调史诗人物与荒诞坐骑的反差。

### 笑点解析：oo

- 长式：一本正经、牵强附会地解释一个本来不好笑的内容，末尾常接“令人忍俊不禁”。
- 短式：`笑点解析：需求又改了`，等于“这事很可笑/很讽刺”。
- 可接`@某人问的`回应“谁问你了”。
- 对严肃事故、伤亡和真实受害者不要使用。

### 卧槽，o

- 源于山泥若观看 Bin 船长名场面时喊`卧槽，Bin`，空耳为`卧槽，冰`。
- 泛化规则：用单字或短称替换，如`卧槽，原/崩/舟/稳`；也可换前缀为`我超/woc`。
- 既能赞精彩操作，也能吐槽突发状况；靠上下文判褒贬。

### 永远滴神

- 原句`乌兹/伍兹，永远滴神`，来自山泥若夸 Uzi。
- 可赞人、物、团队、方案，也可阴阳怪气；`YYDS`是缩写，但书面上原句更有声口。

### 拉胯

- 东北话“拉了胯”经电棍吐槽酒桶操作传播，意为状态差、掉链子、性能不行。
- 正字优先`拉胯`，常见误写`拉垮`。
- 物、人、团队、功能都能修饰；对人用时宜说具体表现而非人格。

### 古神语

- 由电棍音频倒放空耳形成，重点是“不可理解的音节感”，不是稳定外语。
- 常见符号：`阿黑路西`（没处理好）、`欧西给`（AQ音节）、`奥利安费`（别追了）、`哈比下`、`安修，奥克苏恩`、`Pause/PUSH`、`All In`、`Yamato`、`啊米浴说的道理`。
- 只需少量点缀；不要把空耳当真实翻译，也不要连续堆满整段。

### 引流狗 / ylg / 易拉罐

- 指通过切片获得片面信息后，不懂装懂、高强度玩烂梗的人；不等于所有新观众。
- `易拉罐`是为规避平台禁词的替称，`ylg`是缩写。
- 适合内部判断“别玩太烂”；不要随意称呼普通用户。

### 神人

- 源自炫神动态`这个MyGO啊，点进去能看30秒的都算是神人了`。
- 表示异于常人，通常偏贬；可作名词或形容词，衍生`五字神人`。
- 最自然句式：`能X超过30秒的都算神人了`。

### 木柜子

- 最初是针对 MyGO!!!!! 的攻击性谐称，后负面含义弱化，并泛指少女乐队。
- 仅在二次元/乐队语境使用；不要用来攻击国籍或群体。

### oo在哪？

- 因炫神夸`许哥牛逼`后被回`牛逼在哪？`而形成否定评价模板。
- 标准攻防：`稳定在哪？`—`哪哪都稳定`。
- 适合短评论；正式分析应随后给理由。

### 你看你扫的

- 用于评价粗糙、怪异的炫神二创；可泛化为`你看你做的/写的/调的`。
- 语气介于嫌弃与乐在其中，适合熟人作品互损。

### 给我这个oo看笑了

- 评论区反应模板，表示某对象荒诞、低级或意外好笑。
- 变体：`给我这个报错看笑了`、`这个需求给我看笑了`。

### 哇这个好可爱啊

- 可真夸可爱，也常反语调侃丑、怪、猎奇或意义不明的内容。
- 变体：`可爱捏`、`这个可爱`、`地球还要被这种萌物统治多久`。
- 对真人外貌不要用反语羞辱。

### 踩踩背 / CCB

- 来自《豌豆笑传之踩踩背》中大蓝象棒读`老爷爷，我给你踩背来喽`，经二创传播。
- CCB 有两条可独立使用的路径：动作义与定型文。不要只会把任何东西都说成“踩踩背”。

#### 动作义

- 一个东西踩、压、砸到另一个东西上时，可说`给X踩踩背来了`或引用`老爷爷，我给你踩背来喽`。
- 这条用法强调上下叠压的视觉关系，例如`大依赖给主线程踩踩背来了`。

#### `XX笑传之YYY`定型文

- 固定骨架：`XX笑传之YYY`。
- `XX`是被调侃的主题、产品、人物、技术或事件，不要求首字母为 CCB；中文、英文、缩写和中英混写均可。
- `YYY`才是 CCB 槽位，必须满足以下二选一：
  1. **中文式**：恰好三个汉字，逐字拼音首字母依次为 C-C-B。
  2. **英文式**：恰好三个英文词，逐词首字母依次为 C-C-B。
- `YYY`要能概括当前对象的故障、行为或荒诞点；优先语义贴题，其次才是凑首字母。
- 输出时通常直接给完整标题，不补“这是CCB梗”等解释。

中文例子：

| 场景 | 成句 | C-C-B 校验 |
|---|---|---|
| SQL查错表 | `SQL笑传之查错表` | 查 chá / 错 cuò / 表 biǎo |
| 部署传错制品 | `部署笑传之传错包` | 传 chuán / 错 cuò / 包 bāo |
| Git推错版本 | `Git笑传之冲错版` | 冲 chōng / 错 cuò / 版 bǎn |
| 前端层层报错 | `前端笑传之层层崩` | 层 céng / 层 céng / 崩 bēng |
| 文档反复重排 | `排版笑传之重抄版` | 重 chóng / 抄 chāo / 版 bǎn |

英文例子：

| 场景 | 成句 | C-C-B 校验 |
|---|---|---|
| vibe coding中Claude Code坏了 | `vibe coding笑传之Claude Code Broken` | Claude / Code / Broken |
| 上下文崩塌 | `AI agent笑传之Context Collapse Begins` | Context / Collapse / Begins |
| 容器频繁崩溃 | `容器笑传之Container Crashes Badly` | Container / Crashes / Badly |
| 编译器不停抱怨 | `编译笑传之Compiler Complains Bitterly` | Compiler / Complains / Bitterly |

生成步骤：

1. 提取当前话题作为`XX`，如`SQL`、`vibe coding`、`部署`、`前端`。
2. 概括最有节目效果的动作或故障，如“查错表”“Claude Code坏了”。
3. 选择中文三字或英文三词路径，把概括重写为 C-C-B。
